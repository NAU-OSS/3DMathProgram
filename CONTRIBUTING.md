# How to contribute (pull requests, issues, etc.)
Create a pull request to be reviewd and once the code is merged the merger will close the issue.

# Code style or formatting guidelines
Make sure you use camel casing, any other coding formats are fine as long as they're realatively readable.

# Testing requirements
There will be two testing phases and reviews from outsied sources before code is commited.

# Documentation standards
Make sure there is commenting where needed.

# How to report bugs
Create a new issue please!

# How to propose new features
Create a new isssue!

# Community guidelines and expectations
Be kind when reviewing. Review code how you would want your code to be reviewed.
